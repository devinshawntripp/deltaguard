**Fix SCANNER_PATH mismatch** in worker-deployment.yaml line 60:
Change `value: /usr/local/bin/scanner` to `value: /usr/local/bin/scanrook`.
This is a pre-existing bug — the binary was renamed from `scanner` to `scanrook` but the manifest was never updated. The Dockerfile ENV sets the correct value, but the K8s manifest env var overrides it at runtime. Without this fix, the Go worker cannot find the scanner binary and all jobs fail silently.

**Update image tags** in both manifests:
- `deployment.yaml` line 36: Change `image: devintripp/deltaguard-ui:v1.10.0-phase2` to `image: devintripp/deltaguard-ui:v1.10.2`
- `worker-deployment.yaml` line 36: Change `image: devintripp/rust-scanner-worker:v1.9.2` to `image: devintripp/rust-scanner-worker:v1.10.2`

**Commit manifest changes** in scanrook-ui repo:
```bash
cd /Users/devintripp/Desktop/GitHub/scanrook/scanrook-ui
git checkout -b chore/deploy-v1.10.2
git add k8s/deltaguard/deployment.yaml k8s/deltaguard/worker-deployment.yaml
git commit -m "chore: update manifests for v1.10.2 deploy, fix SCANNER_PATH"
git checkout main && git merge chore/deploy-v1.10.2
git push origin main
```
