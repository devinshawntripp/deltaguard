**Gathered:** 2026-03-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix public site navigation so authenticated users aren't unexpectedly redirected away from public pages. The Home page redirect is the primary issue — signed-in users clicking "Home" from docs get sent to /dashboard instead of seeing the home page. Investigate all public pages for similar unwanted redirects.

</domain>

<decisions>
## Implementation Decisions

### Home page for signed-in users
- Remove the auth redirect from `/` — signed-in users should see the marketing page, not be redirected to /dashboard
- Swap the "Sign in" CTA in the page body for "Go to Dashboard" when the user is authenticated
- The top nav already shows "Dashboard" + "Sign out" for signed-in users — that stays as-is
- Keep the /signin redirect to /dashboard — if you're already signed in, redirecting from /signin makes sense

### Nav link targets
- "Install" stays as an external link to https://scanrook.sh — no change needed
- "Home" link stays pointing to "/" — with the redirect removed, it will work correctly
- No new nav links needed — just fix existing behavior
- Claude should audit all public pages for unwanted auth redirects and fix any found (not just /)

### Public vs dashboard context switching
- Current behavior is fine — public nav shows "Dashboard" button for signed-in users, dashboard nav has "Docs" link
- No changes to nav structure — don't add links, don't unify navs
- Two separate nav contexts (public site shell vs dashboard shell) are intentional

### Claude's Discretion
- How to pass auth state to the home page CTA swap (server component prop vs client component check)
- Whether to audit /register or other auth pages for similar redirect issues
- Implementation approach for the redirect removal

</decisions>

## Existing Code Insights

### Reusable Assets
- `PublicSiteShell.tsx`: Already checks auth via `getServerSession()` and passes `isSignedIn` to `PublicNavClient` — same pattern can pass to page content
- `PublicNavClient.tsx`: Already conditionally renders Dashboard/Sign-out vs Sign-in based on `isSignedIn` prop

### Established Patterns
- Auth checks use `getServerSession(authOptions)` at the layout/page level (server-side)
- `redirect()` from `next/navigation` for server-side redirects
- Session validation: `session?.user?.id && !session.revoked`
- Phase 02-02 decision: PublicSiteShell is an async server component

### Integration Points
- `src/app/page.tsx` — Primary fix location (remove redirect, add conditional CTA)
- `src/components/PublicSiteShell.tsx` — Already passes `isSignedIn` to nav; may need to pass to children too
- `src/app/(auth)/signin/page.tsx` — Keep existing redirect (confirmed by user)
